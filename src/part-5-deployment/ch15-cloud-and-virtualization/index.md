# Chapter 15: Cloud and Virtualization

RDMA was born in environments where applications had direct, unmediated access to hardware. A bare-metal HPC cluster running MPI over InfiniBand is the simplest possible deployment model: each node owns its RDMA NIC exclusively, and the verbs API provides a thin abstraction over the hardware. Cloud computing upends every assumption in this model. Resources must be shared among mutually untrusting tenants. Virtual machines and containers must be migrated between physical hosts for maintenance and load balancing. Network configurations must be managed at fleet scale with consistent policies. The hardware-level isolation that RDMA depends on for performance must coexist with the software-defined flexibility that cloud environments demand.

Bringing RDMA to the cloud is therefore not simply a matter of exposing RDMA NICs to virtual machines. It requires solving a set of deeply intertwined problems: How do you partition a single physical NIC among multiple tenants with performance isolation? How do you maintain RDMA connectivity when a VM is live-migrated to a different host? How do you enforce network security policies when the NIC bypasses the host's software networking stack? How do you make RDMA resources discoverable and schedulable in container orchestration platforms like Kubernetes?

This chapter examines the technologies that address these challenges. We begin with **SR-IOV**, the hardware-based NIC virtualization technology that partitions a single physical NIC into multiple virtual functions, each assignable to a different VM with near-native performance. We then explore **virtio and VDPA**, which trade some performance for greater flexibility, including support for live migration. Next, we examine **containers and Kubernetes**, where RDMA device plugins and namespace isolation enable containerized workloads to use RDMA with appropriate resource management. Finally, we survey the **cloud provider implementations** from Microsoft Azure, AWS, Google Cloud, and Oracle Cloud, each of which has developed a distinct approach to delivering RDMA capabilities at scale.

The landscape is evolving rapidly. Technologies that were experimental three years ago -- such as VDPA and RDMA namespace isolation -- are now production-ready. Cloud providers that initially offered RDMA only for specialized HPC instances are extending RDMA-like capabilities to broader instance types. The direction is clear: RDMA is transitioning from a specialized, bare-metal-only technology to a standard cloud infrastructure capability.
